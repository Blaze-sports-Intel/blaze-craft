Use the mobile-craft skill to walk the stack-choice decision tree for: $ARGUMENTS

Walk the five steps:

1. **Native-feature reach** — does the feature set need StoreKit 2, App Intents, Live
   Activities, ARKit, Vision, Apple Pay, HealthKit, HomeKit, CarPlay, rich WidgetKit, Mac
   Catalyst, visionOS, watchOS, tvOS, or Android equivalents? Three or more → strong native
   pull.
2. **Team capability depth** — who's shipped Swift to TestFlight? Kotlin to Play? RN with
   native bridging? Flutter to either store?
3. **Shipping cadence + OTA needs** — how often will it ship? Are OTA updates needed
   (Expo/EAS Update is uniquely strong)?
4. **Single platform vs both** — iOS-only / Android-only / both day one / iOS-first with
   Android fast-follow?
5. **Decision record** — output the chosen runtime, the constraints justifying it,
   re-evaluation triggers, and the implementing specialist (ios-craft /
   react-native-foundations / expo / future-Android).

Hand off to the named implementing specialist. mobile-craft does not write code.
